ur5_2_controller.py¶

class ur5_2_controller.Actuator2(arg_robot_name)[source]¶

This class recieves whether a package is under the end effector of the ur5_2 arm from the rostopic “check_box”, and then the package’s colour through the rostopic “/order_shipping”. It then makes the ur5_2 arm pick up the package place the package in the appropriate bin, and then publishes a shipped message through the rostopic “order_status”.

Parameters:
  • _robot_ns – Name of the ur5 arm
  • _planning_group – Name of the planning group by the ur5 arm
  • _commander – initialized moveit_commander object
  • _robot – moveit_commander.RobotCommander object to get the group names and the current state of the ur5 arm
  • _scene – moveit_commander.PlanningSceneInterface object to get the Rviz scene of the arm
  • _group – moveit_commander.MoveGroupCommander object to move the ur5 arm
  • _display_trajectory_publisher – Publisher handle to publish to /ur5_2/move_group/display_planned_path
  • _exectute_trajectory_client – Simple Action Client object
  • _planning_frame – gets the planning frame of the ur5 arm
  • _eef_link – gets the end effector link of the ur5 arm
  • _group_names – gets the group names of the ur5 arm
  • _box_name – gets the name of the box to be added or removed from the Rviz scene
  • order_status – Publisher handle to publish Dispatch in rostopic “/order_status” once a package is placed on the conveyor belt
  • order_shipping – Publisher handle to publish the colour and id of the package placed on the conveyor belt
  • _computed_plan – Stores computed trajectory by planner
  • _pkg_path – Stores the path of pkg_task5
  • _file_path – Stores the path of the folder which contains the saved yaml trajectory files
  • _curr_state – Gets the current state of the robot

wait_for_state_update(box_is_known=False, box_is_attached=False, timeout=4)[source]¶

Checks to see if Rviz has updated its scene in 4 different ways according to Arguments-

  1. If the Box was added to the scene
  2. If the End Effector has gripped the box
  3. If the End Effector has released the box
  4. If the Box was removed from the scene
Parameters:
  • box_is_known – True if there is supposed to be a box in the scene, False if there is supposed to be no box in the scene
  • box_is_attached – True if the End Effector is supposed to attach to the box, False if the End Effector is supposed to detach the box
  • timeout – No. Of seconds given to check if the Rviz scene has updated or not
Returns:

Value indicating the success of Rviz updating its scene according to the following parameters
Return type:

boolean
